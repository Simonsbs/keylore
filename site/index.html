Stop giving raw secrets to AI coding tools.

Broker secrets for AI coding tools instead of leaking tokens through .env.

KeyLore is a credential broker for Codex, Gemini CLI, Claude CLI, and other MCP clients. It stores raw secrets separately from AI-facing metadata, lets the agent discover the right credential by context, and brokers the live authenticated action without returning the token value.

Why KeyLore exists

AI agents need credentials to do useful work, but handing them raw tokens is a bad security model. KeyLore gives the agent metadata and a brokered path instead.

Fast path

npm install -g @simonsbs/keylore
keylore-http start
# open http://127.0.0.1:8787/

The real problem

.env files work for apps. They break down for agents.

Process-wide secret exposure

A `.env` file gives the process every secret it can read. That is a poor fit for exploratory AI tools that inspect files, run shell commands, and chain tools dynamically.

Weak intent signals

Token names rarely explain what access is safe, when the token should be used, or what domains and operations actually match the task at hand.

Easy to leak

Environment variables spill into shell output, copied configs, logs, debug dumps, and agent sessions. Once secrets are in the process, blast radius expands fast.

How KeyLore works

Separate the secret, the explanation, and the action.

1

Store the real token

KeyLore keeps the raw secret in a dedicated store. Local mode uses an encrypted local store; advanced deployments can switch to external secret backends later.

2

Add human and LLM context

You explain what the credential is for, when to use it, what domains it can reach, and what the agent must never do with it.

3

Let the agent search metadata

Through MCP, the agent searches metadata only. It does not need raw secret material to choose the right credential.

4

Broker the live request

When access is needed, KeyLore enforces policy and performs the live authenticated call on the agent’s behalf.

Why KeyLore beats plain .env

Use brokered credential access, not a process-wide secret dump.

Plain .env

  • Every process secret is readable if the agent can inspect env
  • No meaningful context beyond a token name
  • No brokered access boundary
  • Easy to leak through shell, logs, or copied files
  • Hard to explain to security reviewers

Why KeyLore over .env for AI agents

  • Raw tokens are stored separately; agents see metadata and brokered access only
  • Per-request access goes through the broker instead of blanket env exposure
  • Agents choose credentials by service, domain, operation, and context fit
  • Works with Codex, Gemini CLI, Claude CLI, and generic MCP clients
  • Local-first install is simple, with stronger controls available later

KeyLore is not just a secret store. It is a secret store plus contextual retrieval plus governed execution for AI tools.
