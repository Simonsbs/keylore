Universal pattern

The same four-step model across clients.

1

Store the token in KeyLore

Do not hand the client a raw secret through env vars if KeyLore is meant to be the security boundary.

2

Add context

Use human context and LLM context so the client can select the token by service, domain, and operation.

3

Connect the MCP client

Register KeyLore as an MCP server using the client’s normal config mechanism.

4

Use KeyLore only

Prompt the agent to search KeyLore and request brokered access instead of inspecting your filesystem or shell.
