Home / Security

KeyLore separates raw secret material from AI-facing metadata and brokers access on request, so models and prompts never need to contain token values.

Principle 1

Separate secrets from metadata

Raw secrets live in a protected store. Searchable metadata is what the agent sees and reasons over.

Principle 2

Broker access instead of returning tokens

KeyLore performs the live authenticated call or governs the operation instead of handing the agent the token value.

Principle 3

Reduce blast radius

Keeping secrets out of prompts, shell env, and casual process inspection makes leaks and misuse easier to contain.

Why not .env?

Because .env solves distribution, not control.

Environment variables are process-wide. Any tool, shell, or agent can read and echo them, leaking through logs, stack traces, prompts, and copied configs. That is a weak default for agentic systems.
