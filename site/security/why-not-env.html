Home / Security / Why not .env

The issue

.env distributes secrets. It does not govern them.

Process-wide exposure

Any tool that can inspect the environment can read the secret, whether it needs it or not.

Weak semantics

The variable name rarely explains what the secret is for or when an agent should avoid it.

Easy leakage path

Shells, logs, crashes, copied configs, and ad hoc debugging can all surface environment-based secrets.

Better model

Use context plus brokered access instead.

KeyLore stores the raw token separately, exposes metadata that helps the model choose the right credential, and brokers the live access request so the token value never has to enter the agent workflow.
