Core workflow

The UI is built around one path: add token, test token, connect AI tool.

1

Save token

Store the real secret and give it a unique token key, service name, allowed domains, and risk profile.

2

Write context

User context explains the token for people. LLM context tells the agent when it should use the token.

3

Preview what the AI sees

KeyLore shows the metadata record that will be visible over MCP so you can confirm the token itself is absent.

4

Check this token

Run a brokered live test such as a GitHub rate-limit check to verify the secret, policy, and target domain all line up.

Next step

Connect MCP

Use the client-specific connection flow after the token passes its brokered test.
